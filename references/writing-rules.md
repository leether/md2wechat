# 写作规则摘要

> 加载时机：Step 0 改写时参考。完整规则见 `references/khazix-writer/rules.json`。
> 方法论来源：khazix-writer（数字生命卡兹克），人格覆盖见 `references/gotchas.md` #G01–#G02。

---

## L1 硬性规则（违规阻止渲染）

| 规则 | 说明 |
|------|------|
| 禁用词 | 「首先、其次、最后」「总之、综上所述」「值得注意的是」等套话 |
| 禁用标点 | 破折号 `——`、中文双引号 `""`（需替换为 `，` 和「」） |
| 结构套话 | 「以下是...""本文将会...""总结一下..」等冗余开头 |
| 空泛工具名 | 只说「有个工具」不说具体名字 |
| 超长段落 | 单段 > 300 字（微信阅读体验差） |

---

## L2 风格一致性（违规输出警告）

| 规则 | 说明 |
|------|------|
| 宏大叙事开头 | 「随着 AI 的发展...」「在当今社会...」→ 直接从现象/悬念开头 |
| 口语化不足 | 缺少「坦率的讲」「说真的」「回到xxx这块」等转场 |
| 句长节奏单一 | 长短句交替，避免连续 5 句以上同长度 |
| 情绪标点缺失 | 适当使用 `！？…` 增强语气（但不滥用） |
| 过度加粗 | 全文加粗 > 10 处 → 稀释重点 |

---

## GEO 合规（L1 阻止 / L2 警告）

| 检查项 | L1（阻止） | L2（警告） |
|--------|------------|------------|
| `summary:` 缺失 | ✅ | — |
| 零结构化（无 H2 / 无卡片 / 无列表） | ✅ | — |
| summary 无数据/结论 | — | ✅ |
| H2 标题模糊（无具体信息） | — | ✅ |
| 零引用块 / 零卡片 | — | ✅ |

---

## MD 排版语法（改写时直接嵌入）

| 语法 | 用途 |
|------|------|
| `## H2 标题` | 橙色圆角区块，章节标记 |
| `:::wechat-card` + `title:` + `tone:` | 深色/浅色卡片 |
| `> 引用文字` | 引用块，左侧 3px 竖线 |
| `![alt](src)` | 正文插图，居中圆角 |
| `:::wechat-image` + `src:` + `caption:` | 带说明的文字图 |
| `` ``` <lang> `` ... `` ``` `` | 围栏代码块，深色背景 |
| `| 列 | 列 |` | 数据表格，橙色表头 |

> ⚠️ 卡片内部不支持表格。`:::wechat-card` 里的 `|...|` 不会被解析。

---

## 标题与摘要的 GEO 设计

- **标题**：悬念/反常识 + 埋入 1 个核心搜索词（副标题结构）
- **`summary:`**：写成「AI 可直接引用的答案片段」— 含核心结论 + 具体论据/数据
- **H2 标题**：每个 H2 覆盖一个用户可能搜索的具体问题
